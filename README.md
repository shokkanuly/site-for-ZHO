# Temirtau Volunteer Platform (TWA)

This is a Telegram Web App (TWA) designed for coordinating volunteering events, social drives, and eco-initiatives (like Jasyl El, Taza Qazaqstan, and Qarmet actions) in the city of Temirtau, Kazakhstan. 

The application is structured to load instantly on 3G mobile networks, using a cyber-minimalist/neo-brutalist technical aesthetic and featuring a fully asynchronous FastAPI backend with a Redis-supported WebSocket coordination chat.

---

## Technical Stack
- **Backend:** Python 3.11+, FastAPI (Async), SQLAlchemy (Async), PostgreSQL, Redis.
- **Frontend:** React, Vite, CSS (vanilla cyber-minimalism styling), Telegram Web App SDK.
- **Geolocation:** Yandex Maps API with a custom interactive geogrid fallback for offline/browser testing.

---

## Directory Structure
- `docker-compose.yml` - PostgreSQL (5432) and Redis (6379) configurations.
- `backend/` - Async Python code, REST API endpoints, WebSockets Pub/Sub system.
- `frontend/` - React frontend with Vite build system.

---

## Setup & Running Locally

### 1. Database & Cache Services (Docker Compose)
Launch local PostgreSQL and Redis containers:
```bash
docker-compose up -d
```

### 2. Backend Server Setup
From the project root:
```bash
# Navigate to backend
cd backend

# Create and activate Python virtual environment
python3 -m venv venv
source venv/bin/activate

# Install async dependencies
pip install -r requirements.txt

# Run backend development server (starts on http://localhost:8000)
python main.py
```
*Note: The database tables are automatically initialized on server startup using the SQLAlchemy schema metadata.*

### 3. Frontend Client Setup
From the project root (in a separate terminal):
```bash
# Navigate to frontend
cd frontend

# Install package dependencies
npm install

# Run Vite dev server (starts on http://localhost:5173)
npm run dev
```

## Local Verification & Simulation Features
Since Telegram Web Apps require verification signatures generated by the Telegram bot container, testing inside normal web browsers can be difficult. To solve this, the application features an automatic **Browser Simulator**:
1. When loaded outside of Telegram, the header displays `EMULATOR ACTIVE`.
2. In the **Профиль** (Profile) tab, a toggle list is available allowing you to switch characters:
   - **Aibek (Organizer):** Allows creating initiatives, viewing rosters, and clicking "Подтвердить присутствие" (Check-in) to award rating points.
   - **Anna / Daniyar (Volunteers):** Allows listing events, choosing locations on the interactive map grid, registering for initiatives, and talking in chat.
3. Open two browser windows side-by-side, log in as different users, register for an initiative, and select the **Координационный чат** (Coordination Chat) to verify real-time messaging synchronizing instantly over the Redis Pub/Sub backend.
